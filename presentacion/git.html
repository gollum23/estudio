¿Qué es un control de versiones?

Es un sistema que registra los cambios realizados sobre un archivo o conjunto de archivos a lo largo del tiempo, para poder recuperar versiones específicas posteriormente.

El sistema de control de versiones sirve para casi cualquier tipo de archivo que se encuentre en un computador.

Sistema de control de versiones locales

El método de control de versiones de muchas personas es copiar el archivo a otra carpeta probablemente indicando la fecha de modificación del archivo, aunque este sistema es muy sencillo, tambien es suceptible a errores.

Para hacer frente a este problema, los programadores desarrollaron VCS(Version Control System) locales que contienen una base de datos en la que se registran los cambios registrados en los diferentes archivos.

Sistema de control de versiones centralizados

El siguiente gran problema es la gente que necesita colaborar con otras personas en un documento, para solventar este problema, se desarrollaron los sistemas de control de versiones centralizados (CVCS), dentro de estos sistemas encontramos el conocido Subversión y Perforce

Esta configuración presenta una gran desventaja, si el servidor centralizado presenta una falla durante una hora, entonces en ese lapso de tiempo nadie podra colaborar o guardar cambios en versionados.

Sistema de control de versiones distribuidos

Aquí es donde entra a participar sistemas como Git, Mercurial, Bazzar. Los clientes no sólo descargan la última versión de los archivos, replican completamente el repositorio. Así, si un servidor muere, y estos sistemas estabn colaborando a través de el, cualquiera de los repositorios de los clientes puede copiarse en el servidor para restaurarlo.

Cada vez que se descarga una instantánea, en realidad se hace una copia de seguridad completa de todos los datos.

Configuración inicial

Git trae una herramienta de configuración llamada git config que permite consultar y establecer las variables de configuración del funcionamiento de git

Tu identidad

lo primero que se debe configurar es establecer el nombre de usuario y el correo electronico, es necesario ya que la confirmaciones de cambios usan esta información

                $ git config --global user.name "Diego Forero"
                $ git config --global user.email "gollum23@gmail.com"

Esto solo es necesario hacerlo una vez

Para comprobar tu configuracion

                $ git config --list

Acceder a la ayuda

Git provee una funcion para acceder a la ayuda de los comandos usudos en git y se puede acceder de la siguiente manera

                $ git help -comando-
                o
                $ git -comando- --help

Para acceder a la ayuda general de git solo se digita git en la terminal y mostrara la ayuda.

Comandos básicos

Inicializar un repositorio

$ git init

Clonar un repositorio

$ git clone -url-

Status del repositorio ver cambios preparados y no preparados

$ git status

Comandos básicos

Seguimiento a nuevos archivos y preparación de archivos

$ git add -nombredearchivo-

Confirmar cambios

$ git commit -m "Comentario"

Confirmar cambios sin necesidad de prepararlos archivos

$ git commit -a -m "Comentario"
